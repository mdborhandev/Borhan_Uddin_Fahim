Entity Framework Core Performance Tuning

9 min read · Mar 2026

Entity Framework Core is a powerful ORM, but it's easy to write queries that perform poorly. Over the years, I've compiled a set of essential tuning tips that make a significant difference in production.

Eager Loading vs Explicit Loading

Use eager loading (.Include) when you always need related data. Use explicit loading when you need it conditionally. Never use lazy loading in production — it leads to the infamous N+1 query problem.

// Good: Eager loading
var orders = await context.Orders
    .Include(o => o.Items)
    .ThenInclude(i => i.Product)
    .ToListAsync();

// Better: Projection
var orderDtos = await context.Orders
    .Select(o => new OrderDto { Id = o.Id, Total = o.Items.Sum(i => i.Price) })
    .ToListAsync();

Compiled Queries

For frequently executed queries, use compiled queries. EF Core caches the query plan, eliminating the need to compile the expression tree on every execution. I use this for all hot-path queries in my applications.

Batch Operations

Use ExecuteUpdate and ExecuteDelete for bulk operations instead of loading entities into memory. This has been a game-changer for performance in my SaaS applications.

These optimizations have reduced API response times by up to 80% in production systems.
