Optimizing Database Performance in SaaS Apps

6 min read · Apr 2026

Database performance is critical for SaaS applications. As your user base grows, poorly optimized queries can bring your application to a crawl. In this article, I'll share the indexing strategies, query optimization techniques, and database design patterns I use in production.

Indexing Strategy

Not all indexes are created equal. I use composite indexes for queries with multiple WHERE conditions, partial indexes for filtered queries, and covering indexes to avoid table lookups. The key is to analyze your query patterns first — never guess which indexes you need.

-- Composite index for common query pattern
CREATE INDEX idx_tenants_status_date
ON orders (tenant_id, status, created_at DESC)
WHERE status IN ('pending', 'processing');

Connection Pooling

In SaaS applications, connection management is crucial. I configure Npgsql connection pooling carefully, using multiple pools for different workload types. Read replicas handle reporting queries while the primary handles writes.

Query Optimization

I use EXPLAIN ANALYZE religiously to understand query execution plans. Common issues I find include sequential scans on large tables, unnecessary joins, and N+1 query problems in Entity Framework Core. Using projected queries and batch loading solves most of these.

These techniques have helped me keep database response times under 50ms even with hundreds of concurrent users in production.
